# Despedida ViaRural Broker v1.0.1

Release de seguridad y hardening posterior a la auditoria de Codex Security.

## Seguridad

- Endurecidas las reglas Firestore para dejar de usar el documento singleton publico `gameState/main`.
- La sincronizacion cloud usa ahora documentos por uid anonimo autenticado: `gameState/{userId}`.
- Las lecturas, escrituras y borrados de estado cloud quedan limitados al owner autenticado.
- Las reglas Firestore validan el wrapper de sincronizacion y la forma base de `GameState` antes de aceptar escrituras.
- El cliente fuerza `updatedBy` con el uid autenticado en lugar de aceptar un identificador local controlable.
- El service worker solo cachea respuestas same-origin, correctas y de tipo esperado.

## Testing

- Anadidas pruebas de regresion para reglas Firestore.
- Anadidas pruebas de regresion para politica de cache del service worker.
- Anadida prueba del path de documento cloud por usuario.

## Notas operativas

- Esta version elimina la exposicion publica del estado cloud compartido.
- La sincronizacion queda aislada por identidad anonima Firebase persistida en el navegador.
- Una futura sincronizacion compartida entre dispositivos deberia introducir un `gameId` o codigo de invitacion no adivinable y reglas Firestore asociadas.
